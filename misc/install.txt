Notas:
-----------------
Por alguna razon, la clase String (string.hpp) a veeces produce memory leaks
en el fichero mark.cpp. Se soluciona desactivando la expansion de funciones
inline en dicho fichero.  El leak se puede comprobar abriendo y cerrando de
la misma un fichero de audio con marcas, utilizando la clase CAudioFile.
Programa de test para MSDOS:

#include <stdlib.h>
#include "caudio.hpp"
#include "alloc.h"

void doit( void )
{
  CAudioFile fi;
  fi.open("marked.aho");
  fi.close();
}

int main( int argc, char * argv[] )
{
  cerr << coreleft() << endl;
  doit();
  cerr << coreleft() << endl;
  return 0;
}
